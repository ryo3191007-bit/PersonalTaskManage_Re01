# タスクマネージャー

業務の計画・実績を記録・分析する個人向けタスク管理 Web アプリです。
予定と実績の比較、中断/再開セッションによる正確な実働時間の計測、遅延要因の統計分析など、自己改善を支援する機能を備えています。

---

## 主な機能

### タスク管理
- タスクの作成・編集・削除
- ステータス管理（未着手 / 進行中 / 中断 / 完了）
- 親子タスク（階層ツリー表示）
- カテゴリ・優先度の設定
- 予定時刻・実績時刻の両方を記録
- 中断・再開ボタンによるセッション単位の実働時間計測
- 開始遅延/早期・所要時間超過/短縮の要因記録
- CSV エクスポート / テキストレポートエクスポート
- 当日の作業負荷バー（設定上限との比較）

### 定常タスク管理
- 繰り返しタスクのグループ管理（日次 / 週次）
- 期間内のタスクを一括生成
- 未完了タスクへの一括更新

### カレンダービュー
- 日・週・月の 3 ビュー
- 時間グリッド（24 時間表示）
- セッションの可視化（中断/再開インジケーター）
- 予定 vs 実績のオーバーレイ表示
- カレンダー上からのタスク削除（ホバーでゴミ箱アイコン表示、2 回クリックで削除）

### 分析ダッシュボード
- 完了率・総実績時間 KPI
- ステータス分布ドーナツチャート
- カテゴリ別作業時間バーチャート
- 月次計画 vs 実績の比較グラフ
- 開始タイミング分析（遅延 / オンタイム / 早期の要因ランキング）
- 所要時間分析（超過 / 適切 / 短縮の要因ランキング）

### 設定
- カテゴリの作成・編集・削除（カラーピッカー付き）
- ブラウザ通知の許可設定

### 認証
- Supabase メール/パスワード認証
- ユーザーごとのデータ分離（RLS）

## 開発環境構築手順

### 前提条件

- Node.js 18 以上
- npm 9 以上
- Supabase アカウント（無料プランで可）

### 1. リポジトリのクローン

```bash
git clone <repository-url>
cd project
```

### 2. 依存パッケージのインストール

```bash
npm install
```

### 3. Supabase プロジェクトの作成

1. [Supabase](https://supabase.com) にアクセスしてプロジェクトを作成
2. ダッシュボードの **Project Settings > API** から URL と anon key をコピー

### 4. 環境変数の設定

```bash
cp .env.example .env   # .env.example がない場合は直接 .env を作成
```

`.env` に Supabase の URL と anon key を記入する。

### 5. データベースのセットアップ

`supabase/migrations/` 内のファイルを番号順に Supabase の SQL Editor で実行する。

| 順序 | ファイル | 内容 |
|---|---|---|
| 1 | `20260507120154_create_tasks_schema.sql` | task_categories・tasks テーブル + RLS |
| 2 | `20260507124519_add_actual_times_to_tasks.sql` | actual_start・actual_end カラム追加 |
| 3 | `20260507132400_fix_difficulty_check_allow_zero.sql` | difficulty 制約修正 |
| 4 | `20260507135810_fix_update_updated_at_search_path.sql` | トリガーセキュリティ修正 |
| 5 | `20260507142514_add_priority_to_tasks.sql` | priority カラム追加 |
| 6 | `20260508015438_add_estimate_factors_to_tasks.sql` | 要因カラム + track_actual 追加 |
| 7 | `20260508033304_add_track_actual_to_tasks.sql` | track_actual デフォルト値修正 |
| 8 | `20260508034739_add_recurrence_groups.sql` | recurrence_groups テーブル + RLS |
| 9 | `20260508040152_add_track_actual_to_recurrence_groups.sql` | グループへ track_actual 追加 |
| 10 | `20260508052021_fix_update_recurrence_groups_updated_at_security.sql` | グループトリガーセキュリティ修正 |
| 11 | `20260512115920_add_task_sessions_and_suspended_status.sql` | task_sessions テーブル + suspended ステータス |

### 6. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで `http://localhost:5173` を開く。

## 使い方

### アプリの起動

1. ブラウザで公開 URL にアクセスする
2. ログイン画面でメールアドレスとパスワードを入力して「ログイン」を押す
   - アカウントがない場合は「サインアップ」タブで新規登録する（メール確認不要）
3. 右上のアイコンでライト/ダークテーマを切り替えられる

---

### タスクの登録

1. サイドバーの「タスク一覧」を開く
2. 右上の「**＋新規タスク**」ボタンを押す
3. フォームに必要な情報を入力して「作成する」を押す

過去に入力したタスク名はタスク名欄に履歴サジェストとして表示される（最大 30 件）。

#### 登録できる内容

| 項目 | 必須 | 説明 |
|---|---|---|
| タスク名 | ◎ | タスクの名称。過去の入力履歴からオートコンプリート可能 |
| 分類 | — | 事前に設定したカテゴリを選択。フォーム内から新規作成も可能 |
| ステータス | — | 未着手 / 進行中 / 中断 / 完了（デフォルト: 未着手） |
| 親タスク | — | 親子関係を持たせる場合に設定。検索ボックスで絞り込み可能 |
| 優先度 | — | 高 / 中 / 低（デフォルト: 中）。一覧では優先度の高い順に表示 |
| 数量 | — | 作業の繰り返し数（デフォルト: 1） |
| 所要時間（分）| — | 1単位あたりの見込み時間（分）。数量 × 所要時間 = 総予定時間 |
| 予定開始日時 | — | 予定の開始日時 |
| 予定終了日時 | — | 数量・所要時間が入力済みの場合は開始日時から自動計算 |
| 予定メモ | — | 予定に関するメモや補足情報 |
| 実績開始/終了 | — | 進行中・完了・中断 のときのみ表示。実績時間を記録 |
| 遅延/前倒し要因 | — | 予定開始と実績開始の差に応じて自動で表示 |
| 見積超過/短縮要因 | — | 予定時間と実績時間の差に応じて自動で表示 |
| 実績メモ | — | 実績に関するメモ |

---

### タスクの編集・削除・ステータス変更

#### ステータス変更（インライン）

タスク一覧の各行右側にあるドロップダウンから直接変更できる。

| 変更前 → 変更後 | 動作 |
|---|---|
| 未着手 → 進行中 | 「開始実績日時」入力ダイアログが表示される |
| 進行中 → 完了 | 「終了実績日時・実績メモ・見積差異要因」入力ダイアログが表示される |
| 未着手 → 完了 | 「開始・終了実績日時・各要因」を一括入力するダイアログが表示される |
| 進行中 → 中断 | 「中断日時」入力ダイアログが表示される（中断時間は所要時間から除外） |
| 中断 → 進行中（再開） | 「再開日時」入力ダイアログが表示される。中断時間が表示される |

#### 編集

各タスクにマウスオーバーすると表示される鉛筆アイコンをクリックすると編集フォームが開く。

#### 削除

各タスクにマウスオーバーすると表示されるゴミ箱アイコンを **2 回クリック**すると削除される（1 回目は確認状態になる）。

#### 中断・再開ボタン

進行中タスクの右に一時停止アイコン（⏸）、中断中タスクの右に再開アイコン（▶）が表示される。

#### フィルター

一覧上部のフィルターバーで以下の条件を組み合わせて絞り込める。

| フィルター | 説明 |
|---|---|
| ステータス | 未着手 / 進行中 / 中断 / 完了 / すべて |
| 分類 | 登録済みカテゴリで絞り込み |
| 日付範囲 | 予定・実績のいずれかが指定期間に重なるタスクを表示 |
| 優先度 | 高 / 中 / 低 / すべて |
| キーワード | タスク名・予定メモへの部分一致検索 |

> デフォルトは「本日」の日付フィルターが設定されている。全期間を表示したい場合は日付をクリアする。

#### 作業負荷バー

一覧上部に本日の使用時間・残り時間・上限時間が表示される。上限はバーの「上限」の数値をクリックして直接編集できる（0.5〜24 時間）。

#### エクスポート

- **CSV 出力**: 全タスクを CSV ファイルとしてダウンロード（Excel 対応の BOM 付き UTF-8）
- **テキスト出力**: 出力項目を選択してテキストレポートを生成。クリップボードにコピーして報告書や日報に貼り付けられる

### カレンダーの参照

1. サイドバーの「**カレンダー**」を開く
2. 右上のボタンで「日」「週」「月」の 3 ビューを切り替える

| ビュー | 特徴 |
|---|---|
| 日 | 24 時間の時間グリッド。セッション（中断/再開）がインジケーターで可視化される |
| 週 | 月〜日（7 日間）を横並び表示 |
| 月 | 月全体を確認。複数日にまたがるタスクはバンド表示 |

- 予定と実績が 5 分以上ずれているタスクは点線のアウトラインで表示される
- 時間スロットをクリックするとその日時が設定済みの「新規タスク」フォームが開く

#### カレンダーからのタスク削除

タスクにマウスオーバーするとゴミ箱アイコンが表示される。**2 回クリック**すると削除される（1 回目は確認状態（赤）になる）。日・週・月ビューの全てで使用可能。

### 分析画面の利用

1. サイドバーの「**分析**」を開く
2. 期間セレクター（全期間 / 月別）で集計範囲を選ぶ
3. タブで分析項目を切り替える

> **集計対象**: 親タスクを持たない（ルート）タスクのみが集計される。

#### 表示内容

| タブ | 指標・グラフ | 説明 |
|---|---|---|
| **概要** | 完了率 | 完了タスク数 ÷ 集計対象タスク数（%） |
| | 総実績時間 | 集計対象タスクの actual_time 合計（時間）|
| | ステータス分布 | 未着手・進行中・中断・完了の件数ドーナツチャート |
| | カテゴリ別作業時間 | カテゴリごとの実績時間バーチャート |
| | 月次作業量（計画 vs 実績） | 月別の予定時間と実績時間の 2 本棒グラフ。月をクリックで日別表示に切り替わる |
| **開始タイミング分析** | 開始タイミング分布 | 遅延 / オンタイム / 早期 の月別積み上げ棒グラフ |
| | 遅延・早期の要因ランキング | 要因ごとのカウントとカテゴリ別ランキング（上位 5 件）|
| **所要時間分析** | 所要時間差分布 | 超過 / 適切 / 短縮 の月別積み上げ棒グラフ |
| | 超過・短縮の要因ランキング | 要因ごとのカウントとカテゴリ別ランキング（上位 5 件）|

### 設定画面の利用

サイドバーの「**設定**」を開く。

#### タスク分類（カテゴリ）の管理

| 操作 | 手順 |
|---|---|
| 追加 | 分類名を入力 → カラーを選択 → 「追加」ボタンを押す |

## 利用上の注意点

### タスク一覧のフィルターについて

- **デフォルトで「本日」の日付フィルターが設定されている。** 新規登録したタスクが一覧に表示されない場合、日付範囲フィルターをクリアして確認すること。
- 日付フィルターは予定日時・実績日時・セッション日時のいずれかが範囲に含まれるタスクを表示する。「予定日時を指定していないタスク」は日付フィルターでは表示されない。

### 実績時間の計算について

- 中断・再開を行うと、中断時間は所要時間から自動的に除外される。タスクカードに表示される実績時間は「開始〜終了の総時間 − 中断時間の合計」になる。
- `actual_time`（実績時間）はセッション（中断/再開の記録）が存在する場合はセッションから自動計算される。フォームで手動入力した値はセッションが存在すると上書きされる場合がある。

### 定常タスクグループの削除について

- **定常タスクグループを削除すると、そのグループから生成されたタスクがすべて削除される。** タスクだけを残してグループを削除することはできない。グループを削除する前にタスクデータが不要であることを確認すること。

### 分析ページの集計対象について

- 分析ページが集計するのは「**親タスクを持たないルートタスク**」のみ。子タスクは集計から除外される（親タスクで二重計上しないため）。
- 「実績時間」が 0 のタスクは、予定時間（`scheduled_start` 〜 `scheduled_end` の差分）で代替集計される。

### ブラウザ通知について

- 通知は `setTimeout` で予約されるため、**ページを閉じると通知は届かない**。タブを開いたまま使用する必要がある。
- 通知をブロックした後に許可に変更する場合は、ブラウザの設定からサイトの通知権限を変更する必要がある（アプリ内のボタンでは再許可できない）。

### 親子タスクについて

- 子タスクを持つ親タスクの所要時間は「子タスクの実績時間合計」で自動算出される。親タスクの実績時間を手動入力しても、子タスクが存在する場合は上書きされる。
- **全ての子タスクが完了すると親タスクが自動的に完了になる**。子タスクの完了ステータスを取り消しても親タスクの完了は自動では取り消されない。

### データの永続性・安全性について

- データは Supabase（クラウド PostgreSQL）に保存される。ブラウザのキャッシュをクリアしてもデータは消えない。
- タスク名の入力履歴（サジェスト）はブラウザの localStorage に保存されるため、ブラウザやデバイスをまたいで共有されない。
